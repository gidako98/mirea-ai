# Security notes

The project uses only synthetic educational data generated by `src/data.py`.
No real personal data, API tokens, passwords or private keys are required.

Rules used in this repository:

- `.env` is ignored by Git and must remain local.
- `configs/.env.example` contains only variable names and safe placeholder values.
- The HTTP service validates input JSON and returns client errors instead of stack traces.
- The demo model is intended for educational use and must not be used for production ticket routing without validation on real, approved data.
